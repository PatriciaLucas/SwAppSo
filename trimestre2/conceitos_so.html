1. Apresentação

Este material organiza o primeiro capítulo do livro, dedicado aos conceitos básicos de sistemas operacionais. A ideia central é explicar por que o sistema operacional existe, quais problemas ele resolve e quais formas ele pode assumir em computadores, servidores, celulares e sistemas embarcados.

Pontos-chave do capítulo

  • Um sistema de computação é formado por hardware, sistema operacional e aplicações.
  • O sistema operacional atua entre os aplicativos e o hardware.
  • Seus objetivos básicos são abstrair recursos e gerenciar recursos.
  • As funcionalidades principais envolvem processador, memória, dispositivos, arquivos e proteção.
  • Há vários tipos de sistemas operacionais, conforme o uso e o ambiente.

2. Sistema de computação

Um sistema de computação é composto por hardware e software. O hardware inclui circuitos e dispositivos físicos, como processador, memória, portas de entrada e saída, teclado, mouse, discos, unidades ópticas e dispositivos USB. O software inclui os programas usados pelo usuário, como editores de texto, navegadores, reprodutores de mídia, editores gráficos e jogos.

Entre os programas de aplicação e o hardware existe uma camada de software chamada sistema operacional. Essa camada organiza o uso dos recursos físicos e oferece aos programas uma forma mais simples e uniforme de usar o computador.

3. Objetivos de um sistema operacional

O hardware possui interfaces complexas e diferentes entre si. A forma de controlar um disco rígido, uma placa de rede ou uma unidade óptica pode variar conforme a tecnologia usada. Sem o sistema operacional, cada programa teria de conhecer muitos detalhes internos de cada dispositivo.

O sistema operacional reduz esse problema por meio de dois objetivos centrais: abstração e gerência.

ObjetivoFunçãoExemplo
AbstraçãoOculta detalhes complexos do hardware e oferece interfaces mais simples.Um programa abre um arquivo sem precisar controlar diretamente o motor, trilhas e blocos do disco.
GerênciaControla a distribuição dos recursos entre programas e usuários.O sistema decide qual tarefa usa o processador, qual área de memória cada aplicação recebe e quando uma impressão será executada.

4. Abstração de recursos

Acessar diretamente os recursos de hardware pode ser trabalhoso. Para abrir um arquivo em disco, por exemplo, seria necessário verificar parâmetros, conferir disponibilidade, acionar o dispositivo, localizar diretórios, posicionar a leitura no local correto e transferir dados para a memória. O sistema operacional simplifica esse processo com abstrações como arquivo, diretório, processo e memória virtual.

Exemplo: arquivo

Para o usuário e para o programador, um arquivo parece uma sequência organizada de dados com operações como abrir, ler, escrever e fechar. Internamente, o sistema operacional traduz essas ações para comandos adequados ao dispositivo real, seja ele disco rígido, SSD, pendrive, CD ou recurso remoto.

Finalidades da abstração

  • Fornecer interfaces de acesso mais simples que as interfaces físicas de baixo nível.
  • Tornar os aplicativos menos dependentes da tecnologia específica do hardware.
  • Padronizar o acesso a dispositivos diferentes por meio de interfaces homogêneas.

5. Gerência de recursos

Os aplicativos usam o hardware para executar suas tarefas: ler dados, salvar arquivos, imprimir documentos, tocar música, acessar a rede e processar informações. Quando várias atividades ocorrem ao mesmo tempo, surgem disputas por processador, memória, impressora, rede e dispositivos de armazenamento.

O sistema operacional define políticas para administrar essas disputas e evitar que uma aplicação prejudique as demais.

Processador

Normalmente há mais tarefas em execução do que processadores disponíveis. O sistema distribui o tempo de CPU entre elas.

Memória RAM

Cada aplicação precisa de uma área de memória. O sistema controla essa distribuição e protege os espaços usados.

Impressora

Como a impressão deve ocorrer de forma exclusiva, o sistema cria uma fila de trabalhos.

Rede e servidor

O sistema deve impedir que um usuário ou processo monopolize recursos e deixe os demais sem atendimento.

6. Funcionalidades de um sistema operacional

Para cumprir os objetivos de abstração e gerência, o sistema operacional atua em várias frentes. As funcionalidades abaixo formam a base de um sistema operacional típico.

6.1 Gerência do processador

Também chamada de gerência de processos, tarefas ou atividades. Seu papel é distribuir a capacidade de processamento entre as aplicações, respeitando prioridades e evitando que uma única aplicação monopolize a CPU. Essa gerência permite a sensação de que cada tarefa possui um processador próprio.

6.2 Gerência de memória

Fornece a cada aplicação uma área de memória própria, isolada das demais e do sistema operacional. Esse isolamento melhora a estabilidade e a segurança. Quando a RAM não é suficiente, o sistema pode ampliar a memória disponível usando armazenamento secundário, criando a abstração de memória virtual.

6.3 Gerência de dispositivos

Cada periférico possui características próprias. A gerência de dispositivos usa drivers e modelos abstratos para permitir que dispositivos semelhantes sejam acessados por interfaces comuns. Assim, diferentes dispositivos de armazenamento podem ser tratados como conjuntos de blocos de dados.

6.4 Gerência de arquivos

Constrói a noção de arquivos e diretórios sobre os dispositivos de armazenamento. Define operações e regras de uso. Em muitos sistemas, a ideia de arquivo também é usada para representar recursos que não são arquivos comuns, como conexões de rede ou informações internas do sistema.

6.5 Gerência de proteção

Define usuários, grupos, permissões, autenticação, autorização, auditoria e controle de acesso. Essa função é essencial em computadores conectados em rede e em sistemas compartilhados por múltiplos usuários.

FuncionalidadeRecurso administradoProblema resolvido
Gerência do processadorCPUDistribuir tempo de processamento entre tarefas.
Gerência de memóriaRAM e memória virtualIsolar aplicações e ampliar a memória disponível.
Gerência de dispositivosPeriféricosPadronizar o acesso a dispositivos diferentes.
Gerência de arquivosArquivos e diretóriosOrganizar dados e definir operações de acesso.
Gerência de proteçãoRecursos do sistemaControlar quem pode acessar o quê.

7. Política e mecanismo

Um ponto importante na construção de sistemas operacionais é separar política de mecanismo.

Política é a decisão de alto nível: por exemplo, qual tarefa deve receber mais processador ou quanta memória uma aplicação pode usar.

Mecanismo é o procedimento de baixo nível que executa a decisão: por exemplo, atribuir memória a um processo, retirar memória, enviar pacote de rede ou alternar a tarefa em execução.

Essa separação torna o sistema mais flexível. É possível alterar regras de decisão sem reescrever toda a parte que se comunica diretamente com o hardware.

8. Categorias de sistemas operacionais

Os sistemas operacionais podem ser classificados conforme uso, ambiente, número de usuários, suporte a rede, requisitos de tempo e tipo de equipamento.

CategoriaCaracterísticas principaisExemplos ou contexto
Batch / loteExecuta tarefas em sequência, geralmente sem interação direta durante o processamento.Processamento de grandes filas de trabalhos.
De redePermite usar recursos de outros computadores e compartilhar recursos locais de forma controlada.Arquivos e impressoras em rede.
DistribuídoDisponibiliza recursos de vários computadores como se fossem parte de um sistema único.Ambientes distribuídos e computação em nuvem.
MultiusuárioIdentifica proprietários de recursos e aplica regras de acesso.Sistemas compartilhados por vários usuários.
ServidorGerencia grandes volumes de recursos, usuários e serviços.Servidores de banco de dados, arquivos e rede.
DesktopVoltado ao uso cotidiano, com interface gráfica, interação e suporte a rede.Windows, macOS e Linux em computadores pessoais.
MóvelProjetado para aparelhos compactos, com atenção a bateria, sensores e conectividade.Android e iOS.
EmbarcadoRoda em hardware limitado e executa aplicações específicas.Automação, eletrônicos domésticos e controladores.
Tempo realPrioriza previsibilidade temporal. O importante é cumprir prazos definidos.Controle industrial, freio ABS, reprodução de mídia.

Sistemas de tempo real podem ser críticos, quando perder um prazo causa consequências graves, ou não críticos, quando o atraso apenas degrada o serviço.

9. Breve histórico dos sistemas operacionais

Os primeiros computadores, no final dos anos 1940, executavam aplicações diretamente sobre o hardware. Ao longo das décadas, os sistemas operacionais cresceram em complexidade e passaram a oferecer compartilhamento de tempo, interfaces gráficas, suporte a rede, mobilidade, virtualização e ambientes distribuídos.

Período / anoMarco histórico
Anos 1940Programas executavam diretamente sobre o hardware; o programador controlava detalhes de entrada, saída e processamento.
Anos 1950Surgem bibliotecas de sistema e programas monitores para auxiliar carga, execução e transferência de dados.
1961CTSS, do MIT, introduz compartilhamento de tempo.
1965IBM lança OS/360; o projeto Multics influencia sistemas posteriores.
1969Ken Thompson e Dennis Ritchie criam a primeira versão do UNIX.
1981Microsoft lança MS-DOS.
1984–1985Mac OS 1.0 e MS-Windows 1.0 popularizam a direção das interfaces gráficas.
1987Surgem Minix e OS/2.
1991Linus Torvalds inicia o desenvolvimento do Linux.
1993–1995Windows NT, FreeBSD, NetBSD, Newton OS e Plan 9 marcam novas linhas de evolução.
1999–2007Virtualização com VMWare, MacOS X, Minix 3, iOS e Android ampliam os ambientes de uso.
2010–2015Windows Phone e Windows 10 representam fases recentes do ecossistema da Microsoft.

10. Exercícios

Questões do capítulo

  1. Quais são os dois principais objetivos de um sistema operacional?
  2. Por que a abstração de recursos é importante para os desenvolvedores de aplicações? Ela tem alguma utilidade para os desenvolvedores do próprio sistema operacional?
  3. A gerência de tarefas permite compartilhar o processador, executando mais de uma aplicação ao mesmo tempo. Identifique as principais vantagens trazidas por essa funcionalidade e os desafios a resolver para implementá-la.
  4. O que caracteriza um sistema operacional de tempo real? Quais são as duas classificações de sistemas operacionais de tempo real e quais são suas diferenças?
  5. Relacione as afirmações aos respectivos tipos de sistemas operacionais: distribuído (D), multiusuário (M), desktop (K), servidor (S), embarcado (E) ou de tempo real (T):
    1. [ ] Deve ter comportamento temporal previsível, com prazos de resposta definidos.
    2. [ ] Sistema operacional usado por uma empresa para executar seu banco de dados corporativo.
    3. [ ] São tipicamente usados em telefones celulares e sistemas eletrônicos dedicados.
    4. [ ] Nesse tipo de sistema, a localização física dos recursos computacionais é transparente para os usuários.
    5. [ ] Todos os recursos do sistema têm proprietários e existem regras controlando seu acesso pelos usuários.
    6. [ ] A gerência de energia é muito importante nesse tipo de sistema.
    7. [ ] Sistema que prioriza a gerência da interface gráfica e a interação com o usuário.
    8. [ ] Construído para gerenciar de forma eficiente grandes volumes de recursos.
    9. [ ] O MacOS X é um exemplo típico desse tipo de sistema.
    10. [ ] Sistemas operacionais compactos, construídos para executar aplicações específicas sobre plataformas com poucos recursos.
  6. Sobre as afirmações a seguir, relativas aos diversos tipos de sistemas operacionais, indique quais são incorretas e justifique:
    1. Em um sistema operacional de tempo real, a rapidez de resposta é menos importante que a previsibilidade do tempo de resposta.
    2. Um sistema operacional multiusuário associa um proprietário a cada recurso do sistema e gerencia as permissões de acesso a esses recursos.
    3. Nos sistemas operacionais de rede, a localização dos recursos é transparente para os usuários.
    4. Um sistema operacional de tempo real deve priorizar as tarefas que interagem com o usuário.
    5. Um sistema operacional embarcado é projetado para operar em hardware com poucos recursos.
